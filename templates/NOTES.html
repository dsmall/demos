Editor in Bootstrap

Here are the changes and additions needed to run the Bootstrap version of editor:

These fall into three groups:

  1. Addition of bootstrap css and js files.
  2. Changes to config.html and __init__.py to allow you to choose between original version of the CodeMirror editor and the new Bootstrap version.
  3. Files with "-bs" in their names are new versions of existing files updated to use Bootstrap. At this stage the previous files are unchanged, allowing us to run the original and Bootstrap versions side by side.
  4. You need a version of CodeMirror later than v2.24.

Big Questions

  • Do we want to keep the original colour scheme (white text on a dark background)? Or just use the out-of-the box Bootstrap style (black text on white/pale grey)?
  • I've used the Bootstrap modal dialogues. There were two options - have the dialogue suddenly appear or have it fade in but also fly down from the top. I much prefer the fade in. The flying down, though initially surprising, is similar to dialogues in OS X Lion.
  • What about button colour? Bootstrap convention has blue as primary, orange as a warning and red for danger. I've largely kept to Rascal's black and red convention but tried Orange and Blue in the query save dialogue and just blue in revert. What do we think?
  • Need to remember to revert Bootstrap to the distribution version and fix difference (location of glyphicons) in an overlay CSS file.

Summing Up

I think the Bootstrap version is an improvement. It's certainly simplified the CSS and made it easier to achieve a consistent look and feel. I think it's desirable to avoid looking like just another Bootstrap site. However this new version of editor looks to me more Rascal than Bootstrap.

The Preferences icon (gear wheel) does nothing yet but will set CodeMirror preferences and save them in editor.conf.

I suspect the next step will be to convert my demos to Bootstrap. This would be a good test of the new editor (as was writing this page) and allow me to try out icon buttons.
